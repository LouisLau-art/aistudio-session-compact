# Compress V2 Design

**Date:** 2026-03-09

## Goal
Replace the current heuristic capsule-based compression stack with a new manual compaction flow built around `story_briefing.json`, `state_snapshot.json`, and `preserved_tail.ndjson`, while keeping transcript export as the default workflow.

## Problem
The existing `compress` implementation is architecturally wrong for the target use case.

It was useful as an exploratory baseline, but it mixes too many concerns into a single artifact:
- old history summarization,
- current-state extraction,
- continuation prompt shaping,
- human-readable handoff rendering.

That produces a weak result for high-context, emotionally loaded, relationship-heavy sessions. It also leaves the system with the wrong abstraction boundary: one `context_capsule.json` that tries to do everything.

Now that `transcript-first` has been proven on the user's real AI Studio session, the project no longer needs to preserve `compress v1` as a default path. Keeping it would only preserve technical debt in a small codebase where a clean reset is cheap.

## Design Principles
- Treat transcript export as the default continuation workflow.
- Treat compression as a manual fallback when transcript export is still too large.
- Prefer a small number of purpose-built artifacts over one overloaded artifact.
- Separate stable background from rolling current state.
- Preserve recent raw turns instead of summarizing everything.
- Keep image/OCR enrichment best-effort and non-blocking.
- Remove `compress v1` write-path debt instead of carrying parallel architectures.

## Recommended Approach
### Product posture
- `transcript` remains the primary workflow.
- `compress` becomes `v2-only`.
- `handoff` becomes `v2-only`.
- `context_capsule.json` is no longer generated by new code.

### New compression artifacts
`compress` should generate:
- `state_snapshot.json`
- `preserved_tail.ndjson`
- `compress.report.json`

`handoff` should read:
- `state_snapshot.json`
- `preserved_tail.ndjson`
- optional `story_briefing.json`

and produce:
- `resume_prompt.md`
- `handoff.md`

## Why Not Keep V1 In Parallel
Parallel generation of both `v1` and `v2` would preserve the wrong mental model and keep maintenance cost alive for no product benefit.

The project is still early:
- the codebase is small,
- users are not depending on a stable public compaction API,
- transcript-first is now the real main path.

This is the cheapest moment to delete the old abstraction instead of migrating around it.

## Alternatives Considered
### 1. Keep `compress v1` and add `v2` alongside it
- Pros: lower migration fear.
- Cons: duplicated semantics, split renderer logic, persistent debt, confusing CLI.
- Rejected.

### 2. Keep transcript-first only and defer compaction entirely
- Pros: least work.
- Cons: does not solve the real long-term need once transcript export is still too large.
- Rejected.

### 3. Replace `compress` and `handoff` with a `v2-only` design
- Pros: clean architecture, clearer artifacts, lower long-term maintenance cost, better fit for manual AI Studio session migration.
- Cons: breaking changes to CLI and tests.
- Chosen.

## Artifact Model
### 1. `story_briefing.json`
Long-lived, manually maintained background context.

Purpose:
- provide stable story background,
- provide person map and framework hints,
- reduce summary drift,
- avoid forcing each compression run to rediscover durable context.

Suggested shape:
```json
{
  "background": {
    "summary": "",
    "emotionalContext": [],
    "workingFrames": []
  },
  "peopleMap": [
    {
      "name": "",
      "relation": "",
      "notes": ""
    }
  ],
  "stableFacts": [],
  "timelineAnchors": []
}
```

### 2. `state_snapshot.json`
Automatically generated current-state artifact.

Purpose:
- summarize what a new session must understand now,
- merge briefing knowledge with recent session evidence,
- separate active state from archive.

Suggested shape:
```json
{
  "version": 2,
  "meta": {
    "createdAt": "",
    "rawPath": "",
    "turnCount": 0,
    "imageCount": 0,
    "chunkCount": 0,
    "mode": "heuristic",
    "modelUsed": "heuristic-local",
    "strategy": "briefing-plus-state-plus-tail"
  },
  "briefing": {
    "sourcePath": "",
    "applied": false
  },
  "background": {},
  "peopleMap": [],
  "stableFacts": [],
  "timelineAnchors": [],
  "currentState": {
    "summary": "",
    "currentObjectives": [],
    "currentStance": [],
    "activeQuestions": [],
    "nextActions": []
  },
  "stableDecisions": [],
  "activeFacts": [],
  "recentTimeline": [],
  "archive": {
    "archivedGoals": [],
    "archivedQuestions": [],
    "archivedFacts": []
  }
}
```

### 3. `preserved_tail.ndjson`
Recent raw turns, uncompressed.

Purpose:
- preserve recent question wording,
- preserve recent emotional framing,
- preserve exact turn ordering,
- avoid losing the latest context in summary compression.

Each line should remain a `SessionTurn` JSON object.

### 4. `compress.report.json`
Operational metadata.

Purpose:
- report what was selected,
- expose tail size,
- expose whether briefing was applied,
- expose image-enrichment warnings without blocking the workflow.

## Tail Selection Strategy
Use a budget-driven recent-tail strategy.

Recommended approach:
- walk backward from the latest turn,
- accumulate turns until a `tailCharsBudget` is reached,
- enforce both:
  - `minTailTurns`,
  - `maxTailTurns`.

Recommended defaults:
- `tailCharsBudget = 120000`
- `minTailTurns = 12`
- `maxTailTurns = 80`

Do not split individual turns. Keep raw turn boundaries intact.

## Image / OCR Failure Policy
Image handling is best-effort only.

Rules:
- image enrichment failure must not block `compress`,
- OCR failure must not block `compress`,
- visual-model failure must not block `compress`,
- missing `images.enriched.jsonl` must not block `compress`,
- `state_snapshot.json` and `preserved_tail.ndjson` must still be produced from text-only data.

Warnings should be recorded in `compress.report.json`, not escalated to a hard failure.

## CLI Changes
### `compress`
Replace old capsule-oriented CLI semantics.

Old shape:
- `compress --out <context_capsule.json>`

New shape:
```bash
bun run dev -- compress \
  --raw ./out/session.raw.ndjson \
  --images ./out/images.enriched.jsonl \
  --briefing ./story_briefing.json \
  --out-dir ./out/compact
```

Outputs in `out-dir`:
- `state_snapshot.json`
- `preserved_tail.ndjson`
- `compress.report.json`

### `handoff`
Replace capsule input with snapshot + tail inputs.

New shape:
```bash
bun run dev -- handoff \
  --snapshot ./out/compact/state_snapshot.json \
  --tail ./out/compact/preserved_tail.ndjson \
  --briefing ./story_briefing.json \
  --out-dir ./out/compact
```

Outputs:
- `resume_prompt.md`
- `handoff.md`

## Codebase Changes
### Remove / replace
- Remove the `compress v1` write path in `src/commands/compress.ts`
- Remove `src/lib/capsule.ts`
- Remove `src/lib/capsule-schema.ts`
- Remove v1-oriented tests once migrated
- Remove `handoff` dependency on `context_capsule.json`

### Reuse where sensible
- `src/lib/briefing.ts` can be upgraded and reused for `story_briefing.json`
- transcript export remains unchanged
- capture and OCR plumbing remain unchanged

## Testing Strategy
### 1. Tail selection tests
- verify budget-driven tail selection,
- verify min/max turn behavior,
- verify latest turns are preserved,
- verify single long turn is not split.

### 2. Snapshot generation tests
- verify briefing is merged correctly,
- verify recent state is prioritized over archive,
- verify archive stays secondary,
- verify active questions and decisions are populated from recent context.

### 3. Failure-tolerance tests
- missing image enrichment still produces output,
- `skipped` / `error` image enrichment still produces output,
- warnings land in `compress.report.json`.

### 4. Handoff rendering tests
- verify `handoff.md` and `resume_prompt.md` render from snapshot + tail,
- verify current state appears before archived context,
- verify recent raw turns are visible in handoff output or clearly referenced.

### 5. End-to-end flow tests
- run `compress` on sample raw data,
- run `handoff` on the generated snapshot/tail,
- verify expected files exist.

## Non-Goals
- automatic compaction during active chatting,
- `replacement_history.json` in phase 1,
- token-precise budgeting,
- preserving `context_capsule.json` generation,
- maintaining old `capsule` semantics as a first-class path.
